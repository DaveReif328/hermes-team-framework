# Hermes Knowledge Base Template

Use this template when creating new knowledge base entries for the Hermes Team Framework.

## Template Structure

```markdown
# [Topic Name]

## Overview
Brief description of the topic (2-3 sentences).

## Key Concepts
- **Concept A**: Definition and relevance
- **Concept B**: Definition and relevance
- **Concept C**: Definition and relevance

## Detailed Information

### Section 1: [Name]
Content for section 1.

### Section 2: [Name]
Content for section 2.

## Relationships
- Related to: [[link to related topic]]
- Depends on: [[link to prerequisite topic]]
- Part of: [[link to parent topic]]

## Sources
1. [Source Name](URL) - Relevance and reliability note
2. [Source Name](URL) - Relevance and reliability note

## Agent Notes
- **Shoshin**: [fresh perspective or pattern observation]
- **Yuval**: [architectural considerations]
- **Satya**: [verification status and confidence]
- **Chase**: [security considerations]
- **Korg**: [synthesis notes and connections]
- **Lathrop**: [performance considerations]
- **Billy**: [implementation notes]

## Status
- **Completeness**: [complete|partial|stub]
- **Last Updated**: [YYYY-MM-DD]
- **Update Frequency**: [daily|weekly|monthly|as-needed]
- **Owner**: [responsible agent or team]

## TODO
- [ ] Item needing verification
- [ ] Gap to fill
- [ ] Area needing expansion

---

*Template Version: 1.0*
*Last Revised: 2026-04-14*
```

## Quality Standards

1. **Accuracy**: All facts must be verifiable; flag unverified claims
2. **Completeness**: Include all relevant information known
3. **Clarity**: Use precise language; avoid jargon without definition
4. **Structure**: Follow template for consistency
5. **Attribution**: Cite sources for all substantive claims
6. **Freshness**: Review and update periodically

## Maintenance

| Activity | Frequency | Responsible |
|----------|-----------|-------------|
| Review | Monthly | Entry owner |
| Update | As-needed | Discovering agent |
| Archive | When obsolete | Korg |

## Integration with Knowledge Graph

All entries should be linked to related topics. Korg maintains the overall knowledge graph structure.

## Access Control

- Default: All agents can read
- Write: Owner + approved agents
- Special: Security-related entries may have restricted access
